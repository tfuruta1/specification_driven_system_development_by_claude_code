# [開始] Claude Code SDD+TDD開発システム v13.0

**SDD（仕様駆動開発）とTDD（テスト駆動開発）を統合した、エンタープライズグレードの開発フレームワーク**

## [目標] 概要

Claude Code v13.0は、CTOとアレックスチーム（SDD+TDDリード、QAドキュメントエンジニア、コード最適化エンジニア、TDDテストエンジニア）がペアプログラミングで開発を進める、高品質かつスケーラブルな開発システムです。循環依存完全解消、モジュール分割アーキテクチャ、100%テストカバレッジを実現しています。

## [特徴] 特徴

### v13.0 新機能
- **循環依存完全解消**: インターフェース抽象化とServiceLocatorパターン
- **モジュール分割アーキテクチャ**: 単一責任原則に基づく4モジュール構成
- **テストファイル統合**: 40ファイル→8ファイル（80%削減）
- **100%テストカバレッジ**: 包括的なテストスイート実装

### アレックスチーム体制
- **CTO（Claude Code）**: プロジェクト管理と意思決定
- **アレックス（SDD+TDDリード）**: 技術主導とアーキテクチャ設計
- **QAドキュメントエンジニア**: 品質保証と文書化
- **コード最適化エンジニア**: パフォーマンス改善とリファクタリング
- **TDDテストエンジニア**: テスト戦略と100%カバレッジ実現

### SDD+TDD統合開発
1. 要件定義書作成
2. 技術設計書作成
3. 実装計画作成
4. 設計レビュー
5. テスト作成（TDD Red）
6. 実装（TDD Green）
7. リファクタリング（TDD Refactor）
8. 最終レビュー

## [開始] クイックスタート

### 1. リポジトリのクローン
```bash
git clone [your-repository-url]
cd [project-directory]
```

### 2. Claude Codeを起動
```bash
claude
```

### 3. アレックスチーム起動
```bash
# 以下のキーワードのいずれか1つで自動起動（OR条件）:
# アレックス、プロジェクト、解析、作成、修正、リファクタリング

# 例: すべて自動でアレックスチーム起動
プロジェクトを開始します
コードを解析してください  
新機能を作成します
バグを修正してください
リファクタリングを実施

# または/auto-modeコマンドで手動起動
/auto-mode start

# 開発フローを選択
1. 新規開発      # SDD+TDDフローで新機能実装
2. 既存解析      # コードベース解析とドキュメント作成
3. バグ修正        # 問題特定と修正
4. リファクタリング  # コード品質改善
```

### 4. 開発開始
```bash
# 現在の状態確認
/auto-mode status

# ペアプログラミング終了
/auto-mode stop
```
